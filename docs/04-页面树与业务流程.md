# 图书馆座位预约系统 — 页面树与业务流程

> 前置文档：`docs/03-PRD-Lite.md`
> 本文件将 PRD Lite 的功能需求转化为页面树和流程关系，直接服务下一步"页面卡与 UI 规范"

---

## 1. 页面总数与页面分布

按照固定页面口径，本期共计 **9 个页面**：

| 端 | 页面数 | 包含页面 |
|----|--------|---------|
| 用户端 | 5 页 | 用户首页、座位列表页、座位详情页、预约提交页、我的预约页 |
| 管理端 | 4 页 | 管理员登录页、预约管理页、座位管理页、统计页 |

> 与 PRD Lite 的页面口径对齐说明（变更原因）：
> - **用户首页**（变更）：替代 PRD 阶段的"切换用户"独立页，将身份选择合并到首页，减少一个非核心页面的路由开销，同时让入口更自然
> - **座位详情页**（从 Modal 改为独立页）：PRD Lite 设计决策 #5 原定为 Modal 弹窗，但经页面树阶段评估，独立页面更便于展示完整的时段占用图、座位设施描述等信息，且避免弹窗在移动端/小屏下的布局问题。**设计决策 #5 同步更新为本决策**
> - **统计页**（纳入本期，提升为 P1）：验收标准明确要求统计指标展示，纳入 Sprint 2 管理端开发

## 2. 用户端页面树

### 2.1 页面层级关系

```
                      ┌──────────────────┐
                      │   用户首页 (/)    │
                      │  身份选择 + 概览  │
                      └────────┬─────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
     ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
     │  座位列表页    │  │ 我的预约页    │  │ 用户端布局     │
     │  /Seats       │  │ /Reservation │  │ (全局导航)     │
     └───────┬───────┘  │  /My         │  └──────────────┘
             │          └──────────────┘
             ▼
     ┌──────────────┐
     │  座位详情页    │
     │  /Seats/      │
     │  Detail/{id}  │
     └───────┬───────┘
             │
             ▼
     ┌──────────────┐
     │  预约提交页    │
     │  /Reservation │
     │  /Create      │
     └──────────────┘
```

> **用户首页的双重视图说明**：路由 `/` 根据 Session 状态返回不同内容。
> - **未选择身份时**：首页展示用户身份选择界面（预设用户卡片列表），引导用户先选择身份。
> - **已选择身份后**：导航栏"首页"返回带数据的仪表板视图（座位概览、今日预约情况、快捷入口）。切换身份通过导航栏右侧的"切换"下拉菜单完成，不需要回到身份选择界面。

### 2.2 页面跳转关系表

| 从 | 到 | 触发条件 | 跳转方式 |
|----|----|---------|---------|
| 用户首页 | 座位列表页 | 点击"预约座位"快捷入口 | 链接跳转 |
| 用户首页 | 我的预约页 | 点击"我的预约"快捷入口 | 链接跳转 |
| 座位列表页 | 座位详情页 | 点击某个座位行 | 链接跳转 `/Seats/Detail/{id}` |
| 座位详情页 | 预约提交页 | 点击"预约此座位"按钮 | 链接跳转 `/Reservation/Create?seatId={id}` |
| 预约提交页 | 我的预约页 | 预约提交成功 | 自动跳转（POST 后 Redirect） |
| 预约提交页 | 座位详情页 | 预约提交失败（冲突） | 停留在当前页，显示错误信息 |
| 我的预约页 | — | 点击"取消预约" | 弹窗确认，确认后刷新当前页 |
| 用户首页/任一用户端页面 | 用户首页 | 点击导航栏"首页" | 链接跳转 `/` |
| 任一用户端页面 | 座位列表页 | 点击导航栏"座位列表" | 链接跳转 `/Seats` |
| 任一用户端页面 | 我的预约页 | 点击导航栏"我的预约" | 链接跳转 `/Reservation/My` |

### 2.3 导航栏结构

用户端布局提供顶部导航栏，在所有用户端页面保持一致：

```
[首页]  [座位列表]  [我的预约]         当前用户: 学生 A ▼ [切换]
```

- **已选择身份时**：导航栏完整显示[首页] [座位列表] [我的预约]，右侧显示当前用户名 + [切换]下拉菜单
- **未选择身份时**：导航栏仅显示[首页]，其余菜单项隐藏。但首页上的"浏览座位"快捷入口可直接跳转到座位列表（只读模式，预约按钮不可用，并提示"请先选择身份"）
- 「切换」下拉菜单列出预设体验用户（学生 A/B/C），点击即切换身份，无需密码。管理员不在此列表，须通过 `/Admin/Login` 独立登录
- 导航栏右侧始终显示当前登录用户名；未选择身份时显示"未登录"

## 4. 用户主链路

### 链路 A：完整预约流程（最常用）

```
用户首页 ──→ 座位列表 ──→ 座位详情 ──→ 预约提交 ──→ 我的预约
 ① 选择身份    ② 筛选楼层   ③ 查看时段    ④ 选时段     ⑤ 确认记录
                (或查看)      占用情况    提交预约      状态:待使用
```

### 链路 B：快速取消流程

```
用户首页 ──→ 我的预约 ──→ 点击"取消预约" ──→ 确认弹窗 ──→ 刷新列表
 ① 选择身份    ② 找到目标   ③ 确认取消      ④ 点"确认"   ⑤ 状态变"已取消"
                预约记录
```

### 链路 C：游客浏览流程（未选择身份）

```
用户首页 ──→ 座位列表
 ① 看到身份   ② 可浏览座位
    选择界面     但无法预约
                 (操作按钮
                 引导到首页
                 选择身份)
```

### 主链路闭环验证

```
Start: 用户访问 /
  → 看到用户首页（含身份选择）
  → 选择"学生 A"身份
  → 首页显示座位概览 + 快捷入口
  → 点击"预约座位" → 座位列表页
  → 选择楼层 → 列表更新
  → 点击某座位 → 座位详情页（独立页面，显示完整时段占用图）
  → 点击"预约此座位" → 预约提交页
  → 选择日期 + 勾选时段 → 点击"提交预约"
  → 冲突检测通过 → 跳转到我的预约页
  → 看到新记录，状态"待使用"
  → 点击"取消预约" → 确认弹窗 → 确认
  → 记录状态变为"已取消"
  End: 全流程闭环 ✅
```

## 5. 管理主链路

### 链路 A：日常管理流程

```
管理员登录页 ──→ 预约管理页 ──→ 座位管理页 ──→ 统计页
 ① 输入账号     ② 查看当天    ③ 维护座位    ④ 查看使用
    密码登录      预约列表      信息          数据
```

### 链路 B：预约处理流程

```
管理员登录页 ──→ 预约管理页
                 ① 默认显示全部预约
                 ② 下拉筛选"待使用"
                 ③ 对已到馆学生 → 点"标记完成"
                 ④ 对违规预约 → 点"取消"
                 ⑤ 筛选"已完成" → 确认当天记录
```

### 主链路闭环验证

```
Start: 访问 /Admin/Login
  → 看到登录表单
  → 输入 admin / admin123 → 登录成功
  → 跳转到预约管理页
  → 看到所有预约列表（可按状态筛选）
  → 找到一条"待使用"预约 → 点击"标记完成"
  → 状态变为"已完成"
  → 侧边栏点击"座位管理"
  → 看到座位列表 → 点击"新增" → 添加一个新座位
  → 新座位出现在列表中
  → 侧边栏点击"统计"
  → 看到今日预约数等数据
  → 点击"退出登录"
  → 跳回到管理员登录页
  End: 全流程闭环 ✅
```

## 6. 状态流转

### 6.1 预约状态流转

```
                     ┌────────────────────────────────┐
                     │                                │
                     ▼                                │
      ┌──────────────┐   管理员标记完成    ┌────────┐  │
      │   待使用      │ ────────────────→ │ 已完成  │  │
      │  (Pending)   │                   │ (Done) │  │
      └──────┬───────┘                   └────────┘  │
             │                                       │
             │ 取消预约                                │
             │ (用户或管理员)                          │
             ▼                                       │
      ┌──────────────┐                               │
      │   已取消      │ ──────────────────────────────┘
      │  (Cancelled) │
      └──────────────┘
```

| 状态 | 含义 | 触发动作 | 触发角色 | 影响 |
|------|------|---------|---------|------|
| 待使用 | 预约已提交，尚未到馆 | 预约提交成功 | 系统自动 | 对应时段座位标记为"占用" |
| 已完成 | 已到馆使用 | 管理员标记"完成" | 管理员 | 记录保留，座位释放 |
| 已取消 | 预约被取消 | 用户或管理员取消 | 用户、管理员 | 记录保留，座位释放 |

### 6.2 页面中的状态体现

| 状态 | 座位列表显示 | 我的预约显示 | 预约管理显示 |
|------|------------|------------|------------|
| 待使用 | 座位标"占用" | 显示"待使用"标签 + 可取消 | 显示"待使用"标签 + 可操作 |
| 已完成 | 座位标"空闲" | 显示"已完成"标签 + 无操作 | 显示"已完成"标签 + 无操作 |
| 已取消 | 座位标"空闲" | 显示"已取消"标签 + 无操作 | 显示"已取消"标签 + 无操作 |
