## C. 修改后文档的新一轮审查

修复旧问题后，对全文进行二次审查，发现以下值得注意的点（均非阻塞问题）：

### 观察 1 — Status 字段类型可考虑枚举

§5a 数据模型中 `Reservation.Status` 定义为 `string`，业务上只有三种取值（待使用/已完成/已取消）。建议在后续实现阶段改用 `enum`，而非 magic string。**不在立项阶段修改，仅提醒下一阶段注意。**

### 观察 2 — "座位使用率"计算公式可更精确

§5 统计页写"座位使用率（已预约 slot / 总 slot 百分比）"。这里的"总 slot"建议在需求分析阶段明确是"当日总 slot"还是"当前时段总 slot"。**当前定义已足够进入下一阶段。**

### 观察 3 — M4 时间仍偏紧

虽然增加了优先级排序，但 M4（第 9-10 天）仍承担：联调 + Bug 修复 + 样式 + Git 整理 + 演示准备。建议在下一阶段做时间规划时，将 **演示材料准备** 分散到 M2/M3 完成（每完成一个功能随手截图），不要堆到 M4。
