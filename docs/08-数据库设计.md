## 1. 数据设计目标

### 1.1 本期目标

设计一个**支撑 MVP 功能运行**的数据库结构，满足：

- 座位管理与展示（列表、筛选、详情）
- 预约提交与冲突检测
- 我的预约与取消
- 管理员登录与管理操作
- 数据通过 Seed Data 初始化，可在本地运行演示

### 1.2 设计约束

| 约束 | 说明 |
|------|------|
| 数据量级 | 座位 < 100 条，预约 < 1000 条，不做分页 |
| 数据库 | SQL Server LocalDB，随 Visual Studio 2022 安装 |
| ORM | Entity Framework Core 8.x Code First |
| 仅 3 张表 | Seat、Reservation、Admin，不引入额外关联表 |
| 无存储过程/触发器 | 全部业务逻辑在 Service 层通过 C# 处理 |
| 密码明文 | 仅限课堂演示，生产环境必须用 bcrypt |

## 2. 核心实体清单

以前序文档（PRD-Lite §11 + 系统设计 §4.2）为准，本期共 **3 个实体**：

| # | 实体 | 表名 | 种类 | 说明 |
|---|------|------|------|------|
| 1 | **Seat** | `Seats` | 主数据 | 座位定义：楼层、区域、编号、是否启用 |
| 2 | **Reservation** | `Reservations` | 业务数据 | 预约记录：关联座位、预约人、时间段、状态 |

## 3. 关系说明

### 3.1 E-R 关系

```
Seat (1) ──────────< (N) Reservation
  Seat.Id ──────── SeatId (FK)

Admin (独立表，不与 Seat/Reservation 直接关联)
```

### 3.2 关系详述

| 父实体 | 子实体 | 关系类型 | 外键 | 说明 |
|--------|--------|---------|------|------|
| Seat | Reservation | 一对多 | `Reservation.SeatId` → `Seat.Id` | 一个座位可有多条预约记录；一条预约记录只对应一个座位 |
| — | — | — | — | Admin 表独立，运营数据不关联 |

### 3.3 导航属性

```csharp
// Seat.cs — 一个座位有多条预约记录
public ICollection<Reservation> Reservations { get; set; }
    = new List<Reservation>();

// Reservation.cs — 一条预约记录对应一个座位
public Seat Seat { get; set; }  // 导航属性
public int SeatId { get; set; } // 外键
```

> **注意**：Admin 表不设导航属性，Admin 与业务数据之间通过 Session 状态关联，而非数据库外键。

## 4. 表结构设计

### 4.1 座位表（Seat → `Seats`）

| # | 字段名 | 类型 | 约束 | 默认值 | 说明 |
|---|--------|------|------|--------|------|
| 1 | `Id` | `int` | PK, 自增 | — | 座位唯一标识 |
| 2 | `Floor` | `nvarchar(20)` | NOT NULL | — | 楼层，如"2 楼"、"3 楼" |
| 3 | `Area` | `nvarchar(50)` | NOT NULL | — | 区域，如"A 区"、"自习区" |
| 4 | `SeatNumber` | `nvarchar(20)` | NOT NULL, UNIQUE | — | 座位编号，如"A-01"（全局唯一） |
| 5 | `IsEnabled` | `bit` | NOT NULL | `true` (1) | 是否启用；false 时前端不可预约 |

**索引：**

| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| `IX_Seat_SeatNumber` | `SeatNumber` | 唯一索引 | 防止重复座位编号 |
| `IX_Seat_Floor_IsEnabled` | `Floor`, `IsEnabled` | 普通索引（复合） | 按楼层筛选可用座位 |

**对应的 Entity 类（EF Core Code First）：**

```csharp
public class Seat
{
    public int Id { get; set; }
    public string Floor { get; set; } = "";
    public string Area { get; set; } = "";
    public string SeatNumber { get; set; } = "";
    public bool IsEnabled { get; set; } = true;

    // 导航属性
    public ICollection<Reservation> Reservations { get; set; } = new List<Reservation>();
}
```

**Fluent API 配置（在 `AppDbContext.OnModelCreating` 中）：**

```csharp
modelBuilder.Entity<Seat>(entity =>
{
    entity.HasIndex(s => s.SeatNumber).IsUnique();
    entity.HasIndex(s => new { s.Floor, s.IsEnabled });
    entity.Property(s => s.Floor).HasMaxLength(20).IsRequired();
    entity.Property(s => s.Area).HasMaxLength(50).IsRequired();
    entity.Property(s => s.SeatNumber).HasMaxLength(20).IsRequired();
});
```

### 4.2 预约表（Reservation → `Reservations`）

| # | 字段名 | 类型 | 约束 | 默认值 | 说明 |
|---|--------|------|------|--------|------|
| 1 | `Id` | `int` | PK, 自增 | — | 预约唯一标识 |
| 2 | `SeatId` | `int` | FK → Seat.Id, NOT NULL | — | 关联的座位 |
| 3 | `UserName` | `nvarchar(50)` | NOT NULL | — | 预约人（对应体验账号用户名） |
| 4 | `StartTime` | `datetime2` | NOT NULL | — | 预约开始时间 |
| 5 | `EndTime` | `datetime2` | NOT NULL | — | 预约结束时间 |
| 6 | `Status` | `nvarchar(20)` | NOT NULL | `'待使用'` | 见 §6 状态设计 |
| 7 | `CreatedAt` | `datetime2` | NOT NULL | `GETDATE()` | 创建时间（数据库生成） |

**索引：**

| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| `IX_Reservation_SeatId_TimeRange` | `SeatId`, `StartTime`, `EndTime`, `Status` | 普通索引（复合） | 冲突检测查询：同一座位同一时段是否存在非取消预约 |
| `IX_Reservation_UserName_CreatedAt` | `UserName`, `CreatedAt DESC` | 普通索引（复合） | "我的预约"列表：按用户 + 时间倒序 |
| `IX_Reservation_Status` | `Status` | 普通索引 | 管理端按状态筛选预约 |
| `IX_Reservation_CreatedAt` | `CreatedAt` | 普通索引 | 统计查询、首页概览数据 |

**对应的 Entity 类：**

```csharp
public class Reservation
{
    public int Id { get; set; }
    public int SeatId { get; set; }
    public string UserName { get; set; } = "";
    public DateTime StartTime { get; set; }
    public DateTime EndTime { get; set; }
    public string Status { get; set; } = "待使用";
    public DateTime CreatedAt { get; set; }

    // 导航属性
    public Seat Seat { get; set; } = null!;
}
```

**Fluent API 配置：**

```csharp
modelBuilder.Entity<Reservation>(entity =>
{
    // 外键
    entity.HasOne(r => r.Seat)
          .WithMany(s => s.Reservations)
          .HasForeignKey(r => r.SeatId)
          .OnDelete(DeleteBehavior.Restrict); // 禁止级联删除

    // 默认值
    entity.Property(r => r.Status)
          .HasMaxLength(20)
          .HasDefaultValue("待使用")
          .IsRequired();
    entity.Property(r => r.CreatedAt)
          .HasDefaultValueSql("GETDATE()");
    entity.Property(r => r.UserName)
          .HasMaxLength(50)
          .IsRequired();

    // 索引
    entity.HasIndex(r => new { r.SeatId, r.StartTime, r.EndTime, r.Status })
          .HasDatabaseName("IX_Reservation_SeatId_TimeRange");
    entity.HasIndex(r => new { r.UserName, r.CreatedAt })
          .IsDescending(false, true)
          .HasDatabaseName("IX_Reservation_UserName_CreatedAt");
    entity.HasIndex(r => r.Status)
          .HasDatabaseName("IX_Reservation_Status");
    entity.HasIndex(r => r.CreatedAt)
          .HasDatabaseName("IX_Reservation_CreatedAt");
});
```

## 5. 关键字段说明

### 5.1 楼层（`Seat.Floor`）

以字符串存储，值为 `"2 楼"`、`"3 楼"` 等。不建楼层主数据表（楼层数量少且固定，通过 Seed Data 初始化即可）。

### 5.2 区域（`Seat.Area`）

以字符串存储，值为 `"A 区"`、`"自习区"`、`"期刊阅览区"` 等。同一个楼层可有多个区域。

### 5.3 座位编号（`Seat.SeatNumber`）

**全局唯一**（非楼层层内唯一）。建议命名规则：`{区域缩写}-{序号}`，如 `A-01`、`B-12`、`ZX-05`。

> **设计决策**：PRD-Lite 原文要求"楼层层内唯一"，但系统设计说明 §4.2 中 Fluent API 配置为全局唯一 `HasIndex(s => s.SeatNumber).IsUnique()`。按较新、较贴近实现的系统设计说明为准（全局唯一索引），且通过推荐命名规则 `{区域缩写}-{序号}` 自然实现楼层层的区分。如需跨楼层唯一，可在命名时包含楼层前缀，如 `2F-A-01`。该决策在 Seed Data 阶段需明确。

### 5.4 时段字段（`Reservation.StartTime` / `Reservation.EndTime`）

存储完整的 `DateTime` 值，包含日期和时间。示例：

| 场景 | StartTime | EndTime |
|------|-----------|---------|
| 预约 2026-07-08 09:00-12:00 | `2026-07-08 09:00:00` | `2026-07-08 12:00:00` |
| 预约 2026-07-09 14:00-17:00 | `2026-07-09 14:00:00` | `2026-07-09 17:00:00` |

> **为什么不用 Date + TimeSlot 两张表？** 因为 MVP 中时段计算简单（08:00-22:00 按 1h 切分），不需要独立的时间维度表。直接存储时间范围（StartTime/EndTime 对）即可满足冲突检测的区间重叠查询，且代码更少、更直观。

### 5.5 预约人（`Reservation.UserName`）

存储体验账号的用户名字符串（如 `"学生 A"`、`"学生 B"`）。不设外键关联（无 User 表），查询直接按字符串匹配。

### 5.6 创建时间（`Reservation.CreatedAt`）

由数据库自动生成（`GETDATE()`），不可手动赋值。用于"我的预约"列表按时间倒序排列，以及统计页的时间范围过滤。

## 6. 状态字段设计

### 6.1 预约状态（`Reservation.Status`）

| 状态值 | 含义 | 触发动作 | 触发角色 | 座位占用影响 |
|--------|------|---------|---------|-------------|
| `待使用` | 预约已提交，尚未到馆使用 | 预约提交成功 | 系统创建 | 对应时段座位标记为"占用" |
| `已完成` | 已到馆使用 | 管理员手动标记"完成" | 管理员 | 座位释放 |
| `已取消` | 预约被取消 | 用户或管理员取消 | 用户、管理员 | 座位释放 |

### 6.2 状态流转图

```
                  ┌─────────────────────────────────────┐
                  │                                     │
                  ▼                                     │
   ┌──────────────┐   管理员标记完成      ┌───────────┐ │
   │   待使用      │ ──────────────────→ │   已完成    │ │
   │  (Pending)   │                     │  (Done)   │ │
   └──────┬───────┘                     └───────────┘ │
          │                                            │
          │ 取消预约（用户或管理员）                    │
          ▼                                            │
   ┌──────────────┐                                    │
   │   已取消      │ ───────────────────────────────────┘
   │  (Cancelled) │
   └──────────────┘
```

### 6.3 状态约束

| 状态 | 可转为 | 约束条件 |
|------|--------|---------|
| 待使用 | 已完成、已取消 | 无时间窗口限制 |
| 已完成 | — | 终态，不可再变化 |
| 已取消 | — | 终态，不可再变化 |

### 6.4 座位"空闲/占用"判断逻辑

座位在列表中的占闲状态**不存储**，而是**实时计算**：

> 当前时间 `t` 落在某条**待使用**预约的 `[StartTime, EndTime)` 区间内 → "占用"；否则 → "空闲"。管理员标记"已完成"后座位立即释放（已完成预约不计入占用）。

```sql
-- 判断座位 @seatId 当前是否被占用的 SQL
SELECT COUNT(*)
FROM Reservations
WHERE SeatId = @seatId
  AND Status = '待使用'             -- 仅"待使用"占用座位，"已完成"和"已取消"均释放
  AND StartTime <= @currentTime     -- 预约已开始
  AND EndTime > @currentTime;       -- 预约未结束（左闭右开）
```

## 7. 查询与筛选字段设计

### 7.1 座位列表与筛选

| 查询场景 | SQL / LINQ 模式 | 使用索引 |
|---------|----------------|---------|
| 按楼层列出可用座位 | `WHERE Floor = @floor AND IsEnabled = 1` | `IX_Seat_Floor_IsEnabled` |
| 获取所有不重复楼层 | `SELECT DISTINCT Floor FROM Seats WHERE IsEnabled = 1` | 全表扫描（数据量小） |
| 座位详情（ById） | `WHERE Id = @id` | 主键查找 |
| 座位当前占闲状态 | `Reservations` 表按 SeatId 和时间范围查询 | `IX_Reservation_SeatId_TimeRange` |

### 7.2 预约提交与冲突检测

**核心约束：同一座位同一时段不可重复有效预约。**

```csharp
// 冲突检测逻辑（ReservationService.CreateAsync）
var conflict = await _context.Reservations.AnyAsync(r =>
    r.SeatId == reservation.SeatId &&         // 同一座位
    r.Status != "已取消" &&                     // 非取消状态的预约
    r.StartTime < reservation.EndTime &&       // 区间重叠检测
    r.EndTime > reservation.StartTime);
```

> 区间重叠公式：`A.Start < B.End AND A.End > B.Start`，适用于任何两个区间是否有交集的判断。
>
> 索引 `IX_Reservation_SeatId_TimeRange`（`SeatId, StartTime, EndTime, Status`）直接覆盖此查询的四个筛选字段。

### 7.3 我的预约（用户端）

查询模式：`WHERE UserName = @user ORDER BY CreatedAt DESC`

索引 `IX_Reservation_UserName_CreatedAt` 按 UserName 筛选 + CreatedAt 降序排列，一次索引扫描即可返回结果。

### 7.4 预约管理（管理端筛选）

| 筛选条件 | SQL WHERE | 使用索引 |
|---------|-----------|---------|
| 全部预约 | `ORDER BY CreatedAt DESC` | `IX_Reservation_CreatedAt` |
| 按状态筛选 | `WHERE Status = @status ORDER BY CreatedAt DESC` | `IX_Reservation_Status` |
| 按日期筛选 | `WHERE CreatedAt >= @today AND CreatedAt < @nextDay` | `IX_Reservation_CreatedAt` |
| 按用户搜索 | `WHERE UserName LIKE '%@keyword%'` | 全表扫描（数据量小） |
| 组合筛选（日期+状态+用户） | 多条件 AND | 各单字段索引分别过滤后取交集 |

> MVP 阶段管理端仅实现按**状态**筛选。日期筛选、用户搜索为预留设计，当前不做 UI。

### 7.5 统计查询（P2，本期不实现）

| 指标 | 查询方式 |
|------|---------|
| 今日预约总数 | `SELECT COUNT(*) FROM Reservations WHERE CreatedAt >= @today` |
| 今日已完成 | `...AND Status = '已完成'` |
| 当前占用座位数 | `SELECT COUNT(DISTINCT SeatId) FROM Reservations WHERE Status = '待使用' AND StartTime <= GETDATE() AND EndTime > GETDATE()` |
| 座位使用率 | 已完成预约 / 总座位数（需定义具体口径） |

## 8. 统计字段与统计口径

### 8.1 首页概览数据

| 字段（HomeIndexViewModel） | 数据来源 | 查询方式 |
|---------------------------|---------|---------|
| `TotalSeats` 总座位数 | `Seat` 表 | `SELECT COUNT(*) FROM Seats WHERE IsEnabled = 1` |
| `AvailableSeats` 当前空闲座位 | `Seat` + `Reservation` | 总座位数 - 当前被占用的座位数（当前时间落在某有效预约内） |
| `TodayReservations` 今日预约数 | `Reservation` 表 | `SELECT COUNT(*) FROM Reservations WHERE CreatedAt >= @today` |

### 8.2 统计页指标（P2）

| 指标 | 口径说明 | 备注 |
|------|---------|------|
| 今日预约数 | CreatedAt 在今天内的预约记录总数 | 包括全部状态 |
| 有效预约数 | Status = '已完成' 且 CreatedAt 在今天内的记录数 | 用于反映实际到馆量 |
| 座位使用率 | (今天内被预约的座位数) / 总可用座位数 | 口径待细化 |
| 历史总预约数 | 全部状态的预约记录总数 | 简单计数 |

> 所有统计数字均**实时查询数据库**，不缓存、不预存。MVP 数据量小，统计数据 < 10ms 返回。如需更复杂的统计页，所有聚合操作放在 Service 层完成，不写存储过程。

## 9. 数据一致性与约束说明

### 9.1 唯一约束

| 表 | 约束 | 违反时的影响 |
|----|------|-------------|
| Seat | `SeatNumber` 全局唯一 | 添加/编辑座位时报唯一约束异常，EF Core 抛 `DbUpdateException` |
| Admin | `Username` 全局唯一 | 添加管理员时同理 |

### 9.2 外键约束

| 父表 | 子表 | 约束 | 删除策略 |
|------|------|------|---------|
| Seat (Id) | Reservation (SeatId) | `FK_Reservation_Seat` | `Restrict`（禁止删除有关联预约的座位） |

**管理端删除座位的约束：** 如果一个座位已有预约记录（任意状态），系统拒绝删除该座位。管理员必须先手动确认该座位没有未完成预约，或先禁用（`IsEnabled = false`）而非删除。

### 9.3 默认值约束

| 表 | 字段 | 默认值 | 生成时机 |
|----|------|--------|---------|
| Reservation | `Status` | `'待使用'` | 应用层 EF Core 默认值 |
| Reservation | `CreatedAt` | `GETDATE()` | 数据库自动生成（插入行时） |
| Seat | `IsEnabled` | `1` (true) | 应用层 C# 属性默认值 |

### 9.4 业务约束（Service 层实现，非数据库约束）

| 约束 | 检查方式 | 违反时的行为 |
|------|---------|-------------|
| 同一座位同一时段不可重复有效预约 | Service 冲突检测（`AnyAsync` 查询） | 返回 `(false, "该时段已被预约")`，不写入数据库 |
| 仅"待使用"状态可取消 | Service 检查 `Status == "待使用"` | 返回 `(false, "只有'待使用'状态的预约可以取消")` |
| 取消/完成操作需记录存在 | Service 检查 `FindAsync(id) != null` | 返回 `(false, "预约记录不存在")` |

### 9.5 数据完整性检查清单（开发时逐项验证）

| # | 检查项 | 验证方式 |
|---|--------|---------|
| 1 | 插入两条同一座位完全重叠时段的预约，第二条被拒绝 | 手动测试 |
| 2 | 插入两条同一座位部分重叠时段的预约（如 09-11 和 10-12），第二条被拒绝 | 手动测试 |
| 3 | 插入两条不同座位同一时段的预约，两条都成功 | 手动测试 |
| 4 | 取消一条预约后，同一座位同一时段可以重新预约 | 手动测试 |
| 5 | 座位禁用后，用户端不可见（或不可预约） | 手动测试 |
| 6 | 删除座位时，如果有关联预约，删除被拒绝 | 自动（外键 Restrict） |

## 附录 A：DDL 汇总（仅供对照，实际由 EF Core 迁移生成）

```sql
-- 座位表
CREATE TABLE [Seats] (
    [Id]           INT            IDENTITY(1,1) NOT NULL,
    [Floor]        NVARCHAR(20)   NOT NULL,
    [Area]         NVARCHAR(50)   NOT NULL,
    [SeatNumber]   NVARCHAR(20)   NOT NULL,
    [IsEnabled]    BIT            NOT NULL DEFAULT 1,
    CONSTRAINT [PK_Seats] PRIMARY KEY ([Id]),
    CONSTRAINT [UQ_Seats_SeatNumber] UNIQUE ([SeatNumber])
);
CREATE INDEX [IX_Seat_Floor_IsEnabled] ON [Seats] ([Floor], [IsEnabled]);

-- 预约表
CREATE TABLE [Reservations] (
    [Id]         INT            IDENTITY(1,1) NOT NULL,
    [SeatId]     INT            NOT NULL,
    [UserName]   NVARCHAR(50)   NOT NULL,
    [StartTime]  DATETIME2      NOT NULL,
    [EndTime]    DATETIME2      NOT NULL,
    [Status]     NVARCHAR(20)   NOT NULL DEFAULT '待使用',
    [CreatedAt]  DATETIME2      NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [PK_Reservations] PRIMARY KEY ([Id]),
    CONSTRAINT [FK_Reservations_Seats] FOREIGN KEY ([SeatId])
        REFERENCES [Seats]([Id]) ON DELETE NO ACTION
);
CREATE INDEX [IX_Reservation_SeatId_TimeRange]
    ON [Reservations] ([SeatId], [StartTime], [EndTime], [Status]);
CREATE INDEX [IX_Reservation_UserName_CreatedAt]
    ON [Reservations] ([UserName], [CreatedAt] DESC);
CREATE INDEX [IX_Reservation_Status] ON [Reservations] ([Status]);
CREATE INDEX [IX_Reservation_CreatedAt] ON [Reservations] ([CreatedAt]);

-- 管理员表
CREATE TABLE [Admins] (
    [Id]       INT            IDENTITY(1,1) NOT NULL,
    [Username] NVARCHAR(50)   NOT NULL,
    [Password] NVARCHAR(100)  NOT NULL,
    CONSTRAINT [PK_Admins] PRIMARY KEY ([Id]),
    CONSTRAINT [UQ_Admins_Username] UNIQUE ([Username])
);
```

## 附录 B：迁移注意事项

| 步骤 | 命令 | 说明 |
|------|------|------|
| 1 | `Add-Migration InitialCreate` | 在 `Package Manager Console` 执行，生成初始迁移文件 |
| 2 | 检查生成的迁移文件 | 确认表名、字段类型、索引、默认值与本文设计一致 |
| 3 | `Update-Database` | 应用到 LocalDB，建表 |
| 4 | 运行程序触发 Seed Data | `Program.cs` 中调用 `SeedData.Initialize()` |

> **坑点提醒**：EF Core 默认的 `OnDelete(DeleteBehavior.Cascade)` 会导致级联删除——删除座位时自动删除关联预约。须在 Fluent API 中显式设置为 `DeleteBehavior.Restrict`，否则会违反 §9.2 的约束。
