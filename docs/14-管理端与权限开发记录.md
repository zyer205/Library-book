## 六、默认补足项 / 当前假设

| 补足项 | 说明 |
|--------|------|
| `GetAllSeatsAsync()` 为管理端新增方法 | doc09 §3.3.3 要求"包含已启用和已禁用"，但原代码 `GetAllAsync()` 只返回启用。新增 `GetAllSeatsAsync()` 修复此差异 |
| TempData success 消息 | doc09 只写了失败时设 `TempData["Error"]`，未写成功时设 `TempData["Success"]`。本轮补足，提升管理端操作反馈一致性 |
| 管理端所有页面共用同一套权限检查模式 | `IsAdmin()` → `HttpContext.Session.GetInt32("IsAdmin") == 1`，与 doc09 一致 |
| `AdminCancelAsync` 不校验 UserName | 管理员可取消任意用户的预约，与 doc09 §3.4 一致 |

## 八、验证测试记录

### 管理端流程序列

```
Step 1:  GET  /Admin/Login                         → 200, login form
Step 2:  POST /Admin/Login (wrong pwd)              → 200, error shown
Step 3:  POST /Admin/Login (admin/admin123)          → Redirect to /Admin/Reservations
Step 4:  GET  /Admin/Reservations                    → 200, reservations table
Step 5:  POST /Admin/Reservations/MarkDone/3         → Redirect to /Admin/Reservations
Step 6:  GET  /Admin/Seats                           → 200, all seats (incl. disabled)
Step 7:  GET  /Admin/Statistics                      → 200, stats page
Step 8:  POST /Admin/Login/Logout                    → Redirect to /Admin/Login
Step 9:  GET  /Admin/Reservations (unauthenticated)  → Redirect to /Admin/Login
```

### 用户端回归验证

```
Step 1:  POST /Account/Switch (学生 A)               → Redirect to /Seats
Step 2:  GET  /Seats/Index                           → 200, seat list
Step 3:  POST /Reservation/Create (seat=1, 15-17)    → Redirect to /Reservation/My
Step 4:  GET  /Reservation/My                        → 200, my reservations
```

**结论**：管理端链路完整可用，用户端主链路未受本轮修改影响。
