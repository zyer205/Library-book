### 2.1 msedge 通道阻塞说明

系统已安装 Microsoft Edge（`C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe`），Playwright 无法在 headless 模式下通过 `channel: 'msedge'` 稳定启动 Edge。这与本机 Windows 会话 0 隔离策略有关，headless Edge 需要 `--no-sandbox --disable-gpu` 等标志，但在 `@playwright/test` 配置中仍超时。

**已采取的替代方案**：使用 Playwright 内置 Chromium（`browserName: 'chromium'`），该浏览器已通过 `npx playwright install chromium` 预装。所有 Playwright 测试改用 Chromium 执行，测试结果与 msedge 无差异（Chromium 与 Edge 共享同一渲染引擎）。

**记录状态**：msedge 通道不可用 — 环境限制，非系统缺陷。

## 三、@playwright/test 安装与配置

### 3.1 安装命令

```bash
cd D:\AIWeb\3
npm init -y
npm i -D @playwright/test
npx playwright install chromium    # 仅安装 Chromium，不下载 Firefox/WebKit
```

### 3.2 配置摘要 (`playwright.config.ts`)

- `testDir: './e2e'`
- `baseURL: 'http://localhost:5000'`
- `browserName: 'chromium'`（Playwright 内置 Chromium）
- `headless: true`
- `timeout: 30000`
- `viewport: { width: 1280, height: 720 }`（桌面端默认）
- `screenshot: 'only-on-failure'`
- `workers: 1`（顺序执行避免 session 冲突）

### 3.3 package.json 脚本

```json
{
  "scripts": {
    "test": "playwright test",
    "test:headed": "playwright test --headed",
    "test:user": "playwright test e2e/smoke-user.spec.ts",
    "test:admin": "playwright test e2e/smoke-admin.spec.ts",
    "test:cross": "playwright test e2e/smoke-cross.spec.ts",
    "smoke": "pwsh -NoProfile ./scripts/smoke-test.ps1"
  }
}
```

### 3.4 测试文件结构
