## D. 审计中的特殊注意事项

### 1. 本轮实际修改 vs 骨架代码

Sprint 2 实际只修改了 **5 个文件**（见 docs/14 §四），其余 Admin 控制器/视图/服务均为 Sprint 0 骨架代码，其正确性已在 Stage 12 审计中确认。本轮审计已重新验证这些骨架代码的管理端功能完整性。

### 2. sprint 2 实质是骨架代码验证期

由于 Sprint 0 已完整实现了管理端的 Controller/Service/View 全部代码（含属性路由、TempData、权限守卫等），Sprint 2 的核心工作实际上是验证这些骨架代码的可用性，而非新编写。docs/14 中"已完整实现"的描述是准确的。

### 3. Build 警告说明

14 个警告全为 CS8625（`null` 字面量→非 null 引用）和 CS8619（`string?`→`string` 返回类型不匹配），分布在 Service 层和 Controller 层。这些是 `Nullable` 启用后的类型注解问题，不影响运行。建议 Sprint 3 T15-07 统一修复。
