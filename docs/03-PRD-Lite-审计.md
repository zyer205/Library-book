## C. 修改后文档的新一轮审查

修复旧问题后，对全文进行二次审查，发现以下值得注意的点（均不阻塞）：

### 观察 1 — 设计决策 #5 的位置与标题略有不匹配

设计决策记录段落标题为"来自前期遗留问题"，但 #5（座位详情交互方式）**不是**前期遗留问题，而是 PRD 阶段新增的设计决策。建议将段落标题改为"设计决策记录"，删除"来自前期遗留问题"的限定语，或另分一个子段落。

**位置**：§14 与 §15 之间的设计决策记录区块。

### 观察 2 — §10 验收标准的措辞未同步更新

§10 验收标准第 3 条仍写"点击座位后弹窗/展开显示详情"，使用"弹窗/展开"二选一写法。现在设计决策已经选定 Modal 弹窗，§10 的措辞可以同步简化为"弹窗显示详情"，去掉"展开"选项。**不过这不影响验收功能的可操作性**，属于一致性打磨。

**位置**：§10.1 验收项 #3。

### 观察 3 — F-U02 交互说明仍保留"展开/弹窗"

§5 F-U02 的交互说明仍写"点击座位行可展开/弹窗显示该座位详情"，与设计决策 #5 的"Modal 弹窗"不完全一致。同样属于一致性打磨。

**这两处不一致（观察 2 + 观察 3）都是同一个根因：F-U02 的描述在决策前写的，决策后没有同步更新。建议批量统一。**
