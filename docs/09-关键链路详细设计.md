#### 2.3.1 用户首页（`HomeController`）

| Action | 路由 | 方法 | 职责 |

| `Index()` | `GET /` | `IActionResult` | 根据 Session 状态返回不同视图 |

**GET / → HomeController.Index() 详细流程：**

```
[HttpGet]
public async Task<IActionResult> Index()
{
    var userName = HttpContext.Session.GetString("UserName");

    if (string.IsNullOrEmpty(userName))
    {
        // 未选择身份 → 返回身份选择视图
        // 预设用户列表在 View 中硬编码
        return View();  // → Views/Home/Index.cshtml (身份选择模式)
    }

    // 已选择身份 → 查询数据, 返回仪表板视图
    var totalSeats = await _seatService.GetEnabledCountAsync();
    var availableSeats = await _seatService.GetAvailableCountAsync();
    var todayReservations = await _reservationService.GetTodayCountAsync();

    var viewModel = new HomeIndexViewModel
    {
        IsLoggedIn = true,
        CurrentUserName = userName,
        TotalSeats = totalSeats,
        AvailableSeats = availableSeats,
        TodayReservations = todayReservations,
        Users = new List<UserCard>  // 预设用户列表（只传给身份选择模式）
    };
    return View(viewModel);  // → Views/Home/Index.cshtml (仪表板模式)
}
```

**边界处理：**
- Session UserName 为空 → 展示身份选择，预设用户包括：学生 A、学生 B、学生 C
- Session UserName 不为空 → 展示仪表板，三个统计数字实时查询
- 预设用户列表每次请求都传递到 View——虽然仅在未登录时展示，但 View 通过 `Model.IsLoggedIn` 判断渲染哪个区块

#### 2.3.2 切换用户（`AccountController`）

| Action | 路由 | 方法 | 职责 |
|--------|------|------|------|
| `Switch(string userName, string returnUrl)` | `POST /Account/Switch` | `IActionResult` | 写入 Session，跳转到 returnUrl 或座位列表 |

**POST /Account/Switch 详细流程：**

```
[HttpPost]
public IActionResult Switch(string userName, string returnUrl = null)
{
    // 1. 校验参数
    if (string.IsNullOrEmpty(userName))
        return RedirectToAction("Index", "Home");

    // 2. 写入 Session
    HttpContext.Session.SetString("UserName", userName);

    // 3. 跳转
    if (!string.IsNullOrEmpty(returnUrl) && Url.IsLocalUrl(returnUrl))
        return Redirect(returnUrl);  // 选完身份回到之前想去的页面

    return RedirectToAction("Index", "Seats");  // 默认跳转到座位列表
}
```

**边界处理：**
- userName 为空或非法 → 不写入 Session，直接跳转首页（自动展示身份选择）
- 不需要验证用户名是否在预设列表中（因为切换用户入口只展示预设列表，不可能输入非法值）
- 切换用户时不清除 `IsAdmin` Session（但管理员不需要切换用户功能，所以不影响）

#### 2.3.3 座位列表（`SeatsController`）

| Action | 路由 | 方法 | 职责 |
|--------|------|------|------|
| `Index(string floor)` | `GET /Seats` 或 `GET /Seats?floor=xxx` | `IActionResult` | 展示座位列表，按楼层筛选 |

**GET /Seats → SeatsController.Index() 详细流程：**

```
[HttpGet]
public async Task<IActionResult> Index(string floor = null)
{
    // 1. 检查身份（未选择身份也能浏览，但不可预约）
    var userName = HttpContext.Session.GetString("UserName");
    var isGuest = string.IsNullOrEmpty(userName);

    // 2. 调用 Service 获取数据
    var seats = await _seatService.GetAllAsync(floor);        // 仅已启用座位
    var floors = await _seatService.GetDistinctFloorsAsync(); // 所有楼层下拉选项

    // 3. Foreach 每一条座位，计算当前占闲状态
    foreach (var seat in seats)
    {
        seat.IsOccupied = await _seatService.IsOccupiedNowAsync(seat.Id);
        // IsOccupied 是 Seat 的扩展属性（不在 Entity 中存储）
    }

    // 4. 拼装 ViewModel 返回
    var viewModel = new SeatsIndexViewModel
    {
        Seats = seats,
        CurrentFloor = floor,
        Floors = floors,
        IsGuest = isGuest,         // View 根据此字段决定是否显示"预约"按钮
        CurrentUserName = userName
    };
    return View(viewModel);  // → Views/Seats/Index.cshtml
}
```

**边界处理：**
- `floor` 为空 → 返回全部楼层座位（或默认楼层）
- `floor` 对应楼层不存在 → 返回空列表（View 展示空状态）
- 未选择身份 → `IsGuest = true`，View 中预约按钮不可点击，提示"请先选择身份"
- 已禁用座位（`IsEnabled = false`）→ Service 层过滤掉，不在列表中出现

> **N+1 查询优化提示（Sprint 0 后处理）：** 当前逐条调用 `IsOccupiedNowAsync(seat.Id)` 对每个座位发起一次独立查询（20 个座位 = 20 次数据库往返）。后续可在 `SeatService` 中新增批量方法 `GetOccupiedSeatIdsAsync()`：`SELECT DISTINCT SeatId FROM Reservations WHERE Status = '待使用' AND StartTime <= @now AND EndTime > @now`，一次查询返回所有被占的 `SeatId` 集合（`HashSet<int>`），然后在内存中赋值 `seat.IsOccupied = occupiedIds.Contains(seat.Id)`。

**GET /Reservation/Create?seatId=3 → ReservationController.Create() 详细流程：**

```
[HttpGet]
public async Task<IActionResult> Create(int seatId)
{
    // 1. 检查身份
    var userName = HttpContext.Session.GetString("UserName");
    if (string.IsNullOrEmpty(userName))
        return RedirectToAction("Index", "Home", new { returnUrl = $"/Reservation/Create?seatId={seatId}" });
        // 未登录 → 回首页选身份，选完后根据 returnUrl 回到预约页

    // 2. 检查座位是否存在且已启用
    var seat = await _seatService.GetByIdAsync(seatId);
    if (seat == null || !seat.IsEnabled)
        return NotFound();  // 座位不存在或已禁用 → 404

    // 3. 生成今日日期之后的时段列表（供 View 渲染勾选框）
    var today = DateTime.Today;
    var timeSlots = GenerateTimeSlots(today);  // 08:00~22:00 共 14 个 slot

    // 4. 拼装 ViewModel
    var viewModel = new ReservationCreateViewModel
    {
        SeatId = seatId,
        SeatNumber = seat.SeatNumber,
        Floor = seat.Floor,
        Area = seat.Area,
        SelectedDate = today,           // 默认今天
        AvailableTimeSlots = timeSlots, // 所有 slot（占用判断在 View 中做）
        UserName = userName             // 当前用户，隐藏字段
    };
    return View(viewModel);  // → Views/Reservation/Create.cshtml
}
```

**POST /Reservation/Create 详细流程（核心链路）：**

```
[HttpPost]
public async Task<IActionResult> Create(ReservationCreateViewModel model)
{
    // 1. 检查身份
    var userName = HttpContext.Session.GetString("UserName");
    if (string.IsNullOrEmpty(userName))
        return RedirectToAction("Index", "Home");

    // 2. 校验 ModelState
    if (!ModelState.IsValid)
    {
        // 必填字段不合法 → 重新生成时段列表，返回当前页显示错误
        model.AvailableTimeSlots = GenerateTimeSlots(model.SelectedDate);
        return View(model);
    }

    // 3. 调用 Service 创建预约（含冲突检测）
    var reservation = new Reservation
    {
        SeatId = model.SeatId,
        UserName = userName,
        StartTime = model.SelectedDate.Add(model.StartTimeSpan),
        EndTime = model.SelectedDate.Add(model.EndTimeSpan),
        Status = "待使用"
    };

    var result = await _reservationService.CreateAsync(reservation);

    // 4. 根据结果分流
    if (!result.Success)
    {
        // 失败：冲突或其他业务错误
        ModelState.AddModelError("", result.ErrorMessage);
        model.AvailableTimeSlots = GenerateTimeSlots(model.SelectedDate);
        return View(model);  // 停留在预约页，显示错误
    }

    // 成功：跳转到我的预约
    TempData["Success"] = "预约成功！";
    return RedirectToAction("My");
}
```

**边界处理：**
- 未选身份 → 重定向到首页（前置检查）
- 座位不存在或已禁用 → 404
- ModelState 不合法（如未选择时段、日期为空）→ 返回当前页显示校验错误
- 冲突检测失败 → 返回当前页，ModelState 显示"该时段已被预约，请重新选择"
- 成功 → TempData 存成功消息，跳转到"我的预约"
- 表单中 `UserName` 不信任前端传入，后端从 Session 重新读取（防止篡改）

**`GetAvailableCountAsync()` 的伪代码：**

```csharp
public async Task<int> GetAvailableCountAsync()
{
    var total = await _context.Seats.CountAsync(s => s.IsEnabled);
    var now = DateTime.Now;

    // 当前被占用的座位数 = 有"待使用"预约且当前时间落在时段内的座位
    var occupied = await _context.Reservations
        .Where(r => r.Status == "待使用"
                 && r.StartTime <= now
                 && r.EndTime > now)
        .Select(r => r.SeatId)
        .Distinct()
        .CountAsync();

    return total - occupied;
}
```

**`GetTimeSlotsAsync(seatId, date)` 的伪代码：**

```csharp
public async Task<List<TimeSlotVM>> GetTimeSlotsAsync(int seatId, DateTime date)
{
    // 1. 获取该座位当天的非取消预约
    var reservations = await _context.Reservations
        .Where(r => r.SeatId == seatId
                 && r.StartTime >= date
                 && r.StartTime < date.AddDays(1)
                 && r.Status != "已取消")
        .ToListAsync();

    // 2. 生成 08:00~22:00 共 14 个 slot（每段 1 小时）
    var slots = new List<TimeSlotVM>();
    for (int hour = 8; hour < 22; hour++)
    {
        var slotStart = date.AddHours(hour);
        var slotEnd = date.AddHours(hour + 1);

        // 3. 判断该 slot 是否被预约占用
        var isOccupied = reservations.Any(r =>
            r.StartTime < slotEnd &&
            r.EndTime > slotStart);

        slots.Add(new TimeSlotVM
        {
            StartTime = slotStart,
            EndTime = slotEnd,
            IsOccupied = isOccupied
        });
    }
    return slots;
}
```

### 2.5 DataAccess 动作

|-------------|----------------|------|
| `GetEnabledCountAsync()` | `_context.Seats.CountAsync(s => s.IsEnabled)` | 简单计数 |
| `GetAvailableCountAsync()` | 见 §2.4.1 伪代码 | 需要两张表关联运算 |
| `GetDistinctFloorsAsync()` | `_context.Seats.Where(s => s.IsEnabled).Select(s => s.Floor).Distinct().ToListAsync()` | 全表扫描，数据量小 |
| `GetAllAsync(floor)` | `_context.Seats.Where(s => s.IsEnabled && (floor == null \|\| s.Floor == floor)).ToListAsync()` | 按楼层筛选 |
| `IsOccupiedNowAsync(seatId)` | `_context.Reservations.AnyAsync(r => r.SeatId == seatId && r.Status == "待使用" && r.StartTime <= now && r.EndTime > now)` | 区间重叠判断 |
| `GetByIdAsync(id)` | `_context.Seats.FindAsync(id)` 或 `FirstOrDefaultAsync(s => s.Id == id)` | 主键查找 |
| `GetTimeSlotsAsync(seatId, date)` | `_context.Reservations.Where(r => r.SeatId == seatId && r.StartTime >= date && r.StartTime < date.AddDays(1) && r.Status != "已取消").ToListAsync()` | 日期范围 + 状态过滤 |
| `CreateAsync(reservation)` | ① `AnyAsync` 冲突检测（见 §7.2 数据库设计）② `_context.Reservations.Add(reservation)` + `SaveChangesAsync` | 先查后写 |
| `GetMyAsync(userName)` | `_context.Reservations.Where(r => r.UserName == userName).OrderByDescending(r => r.CreatedAt).Include(r => r.Seat).ToListAsync()` | 含导航属性 Include |
| `GetTodayCountAsync()` | `_context.Reservations.CountAsync(r => r.CreatedAt >= today && r.CreatedAt < today.AddDays(1))` | 日期范围计数 |
| `CancelAsync(id, userName)` | ① `FindAsync(id)` 查记录 ② 校验状态+权限 ③ `reservation.Status = "已取消"` ④ `SaveChangesAsync` | 读-判断-写 |

### 2.6 数据库变化

| 步骤 | 表 | 操作 | 影响行数 | 事务要求 |
|------|----|------|---------|---------|
| 首页加载 | Seats | SELECT COUNT | 1 | 无 |
| 首页加载 | Reservations | SELECT COUNT | 1 | 无 |
| 座位列表 | Seats | SELECT 多行 | 10~20 | 无 |
| 座位列表 | Reservations | SELECT COUNT 多次（逐条判断占闲） | N 次 × 1 | 读一致性不需要（占闲状态即时变化可接受） |
| 座位详情 | Seats | SELECT 1 行 | 1 | 无 |
| 座位详情 | Reservations | SELECT 多行（当日预约） | 0~14 | 无 |
| **预约提交（核心）** | Reservations | SELECT ANY（冲突检测）→ INSERT 1 行 | 1 写入 | **隐式事务**（`SaveChangesAsync` 自动包裹） |
| 我的预约 | Reservations | SELECT 多行（含 Seat Join） | 0~10 | 无 |
| 取消预约 | Reservations | SELECT 1 行 → UPDATE 1 行 | 1 更新 | **隐式事务**（`SaveChangesAsync` 自动包裹） |

> **冲突检测的并发安全性：** 上述逻辑存在**先查后写**的竞态条件——两个用户同时提交同一座位同一时段的预约，两个冲突检测都返回 `false`，两条记录都会写入。当前阶段接受此风险（课堂演示，并发极低）。如果未来需要严格防重复，可在数据库层加**唯一约束**（如 `SeatId + StartTime + EndTime + Status != '已取消'` 的索引视图过滤），或在 Service 层用 `lock` 语句、分布式锁。

### 2.7 状态流转

参见数据库设计 §6 状态图。链路一涉及的状态变化：

| 操作 | 原状态 | 新状态 | 触发点 | 条件 |
|------|--------|--------|--------|------|
| 预约提交成功 | — | 待使用 | `ReservationService.CreateAsync()` | 冲突检测通过 |
| 用户取消 | 待使用 | 已取消 | `ReservationService.CancelAsync()` | 预约存在 + 状态待使用 + 用户名匹配 |

**座位占闲状态（不持久化，实时计算）：**

```
座位状态 = 当前时间 落入 某条(Status=待使用 AND [StartTime, EndTime)) 的预约？
  ├─ YES → "占用"
  └─ NO  → "空闲"
```

**座位占闲状态变化仅取决于时间流逝和预约状态变更：**

| 事件 | 座位占闲变化 |
|------|------------|
| 预约提交成功（待使用） | 若当前时间在时段内 → 变为占用；若不在 → 不变 |
| 预约到达 StartTime | **未实现**（没有定时任务），通过刷新页面重新计算 |
| 预约结束 / 管理员标记完成 | 取消或完成后，刷新页面重新计算 → 可能变为空闲 |
| 用户取消预约 | 刷新页面重新计算 → 该时段变为空闲 |

> **说明**：座位占闲状态不做实时推送。用户刷新页面或跳转时重新计算。`IsOccupied` 是 ViewModel 中的计算属性，不存储在数据库中。

### 2.8 异常与边界情况

#### 2.8.1 预约冲突（核心业务异常）

| 场景 | 触发条件 | Service 返回 | Controller 行为 | 用户看到 |
|------|---------|-------------|----------------|---------|
| 完全重叠 | 同一座位 09:00-12:00 已有预约，再约 09:00-12:00 | `(false, "该时段已被预约，请重新选择")` | `ModelState.AddModelError`，返回当前 View | 表单上方显示红色错误提示 |

{
    if (!ModelState.IsValid)
        return View(model);

    var admin = await _context.Admins
        .FirstOrDefaultAsync(a => a.Username == model.Username
                               && a.Password == model.Password);

        return View(model);  // 返回表单 + 错误提示

    // 登录成功

### 3.4 Service 职责

链路二复用的 Service 方法已在 §2.4 列出，以下是链路二独享的：

| Service 方法 | 输入 | 输出 | 职责 | 所属 Service |
|-------------|------|------|------|-------------|
| `GetAllAsync(string status)` | 状态筛选 | `List<Reservation>`（含 `Seat` 导航属性） | 返回全部预约，支持按状态筛选 | `IReservationService` |
| `MarkDoneAsync(int id)` | 预约 ID | `(bool, string)` | 校验状态→标记"已完成" | `IReservationService` |
| `AdminCancelAsync(int id)` | 预约 ID | `(bool, string)` | 管理员取消预约（不校验用户名） | `IReservationService` |

**`MarkDoneAsync(id)` 的伪代码：**

```csharp
public async Task<(bool Success, string ErrorMessage)> MarkDoneAsync(int id)
{
    var reservation = await _context.Reservations.FindAsync(id);
    if (reservation == null)
        return (false, "预约记录不存在");

    if (reservation.Status != "待使用")
        return (false, "只有'待使用'状态的预约可以标记完成");

    reservation.Status = "已完成";
    await _context.SaveChangesAsync();
    return (true, null);
}
```

**`AdminCancelAsync(id)` 的伪代码：**

```csharp
public async Task<(bool Success, string ErrorMessage)> AdminCancelAsync(int id)
{
    var reservation = await _context.Reservations.FindAsync(id);
    if (reservation == null)
        return (false, "预约记录不存在");

    if (reservation.Status != "待使用")
        return (false, "只有'待使用'状态的预约可以取消");

    reservation.Status = "已取消";
    await _context.SaveChangesAsync();
    return (true, null);
}
```

> **与用户取消的区别**：`AdminCancelAsync` 不校验 `UserName`，管理员可以取消任意用户的预约。`CancelAsync` 多一个 `userName` 参数，校验 `reservation.UserName == userName`。

### 3.6 数据库变化

| 操作 | 表 | 操作 | 事务要求 |
|------|----|------|---------|
| 登录查询 | Admins | SELECT 1 行 | 无 |
| 预约列表 | Reservations + Seats | SELECT 多行（含 JOIN） | 无 |
| 标记完成 | Reservations | SELECT 1 行 → UPDATE Status | 隐式事务 |
| 管理员取消 | Reservations | SELECT 1 行 → UPDATE Status | 隐式事务 |
| 新增座位 | Seats | INSERT 1 行 | 隐式事务 |
| 编辑座位 | Seats | SELECT 1 行 → UPDATE 多列 | 隐式事务 |
| 启用/禁用 | Seats | SELECT 1 行 → UPDATE IsEnabled | 隐式事务 |

### 3.7 状态流转

| 操作 | 原状态 | 新状态 | 触发点 | 条件 |
|------|--------|--------|--------|------|
| 管理员标记完成 | 待使用 | 已完成 | `ReservationService.MarkDoneAsync()` | 预约存在 + 状态为待使用 |
| 管理员取消 | 待使用 | 已取消 | `ReservationService.AdminCancelAsync()` | 预约存在 + 状态为待使用 |

### 3.8 异常与边界情况

| 场景 | 处理方式 |
|------|---------|
| 未登录直接访问 `/Admin/Reservations` | 重定向到 `/Admin/Login` |
| 未登录直接访问 `/Admin/Seats` | 重定向到 `/Admin/Login` |
| Session 超时后点击操作 | 操作时检查 `IsAdmin`，发现为空 → 重定向到登录页，操作不执行 |
| 标记完成一个已完成的预约 | Service 校验失败，`TempData["Error"]` 显示提示 |
| 取消一个已取消的预约 | 同上 |
| 新增座位时编号重复 | `DbUpdateException`，catch 后转为友好错误消息 |
| 座位管理无数据 | View 展示空状态，附带"新增座位"按钮引导 |
