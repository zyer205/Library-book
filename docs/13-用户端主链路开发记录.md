## 三、发现并解决的问题

### 问题 1：路由冲突 — Admin SeatsController 类名

**现象**：学生登录后跳转到 `/Admin/Seats` 而非 `/Seats`。

**根因**：`AccountController.Switch()` 中 `RedirectToAction("Index", "Seats")` — ASP.NET Core 按命名空间顺序查找，`Admin.SeatsController`（完整命名空间 `LibrarySeatReservation.Web.Controllers.Admin`）比 `SeatsController`（`LibrarySeatReservation.Web.Controllers`）更早匹配。

**修复**：在 `Admin/SeatsController` 类上添加 `[Route("Admin/Seats")]` 属性路由，消除隐式路由歧义。

**验证**：POST `/Account/Switch` 后跳转到 `/Seats`。

### 问题 2：TimeSpan 模型绑定格式

**现象**：预约表单 POST 的 `StartTimeSpan` 和 `EndTimeSpan` 被序列化为 ticks（如 `342000000000`）而非 `"09:00"`。

**根因**：Razor `asp-for` 对 `TimeSpan?` 类型默认使用 ticks 格式输出。

**修复**：在 `Reservation/Create.cshtml` 中将 `asp-for` 改为手动 `<input>` 配合 `value="@Model.StartTimeSpan?.ToString(@"hh\:mm")"`，POST 时提交 `"HH:mm"` 字符串，ASP.NET Core 默认模型绑定可正确解析。

**验证**：POST 后成功写入 DB，不出现"结束时间必须大于开始时间"错误。

## 四、端到端验证数据

### 验证脚本（PowerShell via HttpClient）

```
Step 1: GET  /                        → 200, guest view with 3 user cards
Step 2: POST /Account/Switch          → Redirect to /Seats (user=学生 A)
Step 3: GET  /Seats/Index             → 200, 20 seats displayed
Step 4: POST /Reservation/Create      → Redirect to /Reservation/My (seat 1, 09:00-11:00)
Step 5: POST /Reservation/Create      → Conflict error on same seat+time (duplicate protected)
Step 6: POST /Reservation/Create      → Redirect to /My (seat 2, 14:00-16:00)
```
