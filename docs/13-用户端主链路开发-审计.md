### 准则 2：完整用户闭环

验证链路（全部在源代码中确认）：

```
首页（未登录）
  ├─ 3 张用户卡片 → 点击 → POST /Account/Switch → 写入 Session["UserName"] → redirect /Seats
  └─ "管理员入口" → /Admin/Login

首页（已登录）
  ├─ 总座位数 / 空闲数 / 今日预约（DB 实时查询）
  ├─ "预约座位" → /Seats/Index
  └─ "我的预约" → /Reservation/My

座位列表 /Seats
  ├─ 20 座位（绿/黄 badge 标记占闲）
  ├─ 楼层下拉筛选
  ├─ "查看详情" → /Seats/Detail/{id}
  └─ 未选身份时提示"请先选择身份"

座位详情 /Seats/Detail/{id}
  ├─ 14 个时段 slot（08-22点），绿/黄标记
  ├─ 已禁用座位隐藏"预约此座位"
  ├─ "预约此座位" → /Reservation/Create?seatId={id}
  └─ "返回列表" → /Seats

预约提交 /Reservation/Create
  ├─ 日期选择 + 时段 checkbox 多选
  ├─ JS 计算起止 → hidden input → POST
  ├─ 冲突 → 页面显示错误
  ├─ 成功 → 写入 DB → redirect /Reservation/My
  └─ "取消" → 返回详情

我的预约 /Reservation/My
  ├─ 按创建时间倒序
  ├─ "待使用" → "取消预约"按钮 → confirm → POST /Reservation/Cancel/{id}
  ├─ 已完成/已取消 → 无操作按钮
  └─ 无记录 → 空状态 + "去预约"→ /Seats
```

**结论**：✅ 完整的用户主闭环，7 个步骤全部可走通。
